# ERP 系统五人详细任务分配

> 执行周期：7 天  
> 团队人数：5 人  
> 目标完成度：原型功能的 50%～60%  
> 开发原则：优先保证登录、基础资料、采购、销售、出入库、库存和收付款演示链路；非核心功能可以简化为普通 CRUD。

## 1. 全组最终目标

第 7 天必须可以稳定演示下面这条业务链：

```text
管理员登录
  → 新增商品、客户、供应商、仓库
  → 创建采购单
  → 根据采购单办理入库
  → 查看库存增加和库存流水
  → 创建销售单
  → 根据销售单办理出库
  → 查看库存减少和库存流水
  → 登记客户收款和供应商付款
  → 查看首页统计和三个基础报表
```

如果某项功能不在这条主线上，时间不足时都可以降级或暂时取消。

## 3. 第一天必须统一的技术契约

### 3.1 通用接口返回

```json
{
  "code": "SUCCESS",
  "message": "操作成功",
  "data": {}
}
```

分页数据统一为：

```json
{
  "records": [],
  "total": 0,
  "page": 1,
  "size": 10
}
```

### 3.2 通用字段

所有业务表至少包含：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | BIGINT | 主键 |
| `created_by` | BIGINT | 创建人，可暂时允许为空 |
| `created_at` | DATETIME | 创建时间 |
| `updated_by` | BIGINT | 修改人，可暂时允许为空 |
| `updated_at` | DATETIME | 修改时间 |
| `deleted` | TINYINT | 逻辑删除，0 正常、1 删除 |

### 3.3 通用单据状态

采购单、销售单、入库单、出库单尽量统一使用：

| 状态值 | 中文 | 可执行操作 |
| --- | --- | --- |
| `DRAFT` | 草稿 | 编辑、删除、审核 |
| `APPROVED` | 已审核 | 执行入库或出库、取消 |
| `COMPLETED` | 已完成 | 查看，不再编辑 |
| `CANCELLED` | 已取消 | 查看，不再执行 |

一周项目不做复杂的部分入库、部分出库状态。一次单据直接全部执行，减少联调成本。

### 3.4 下拉选项格式

商品、客户、供应商、仓库等选择框统一返回：

```json
[
  {
    "value": "1",
    "label": "示例名称"
  }
]
```

## 4. 成员 1：公共框架、登录和系统设置

### 4.1 职责边界

成员 1 是公共框架负责人。主要目标不是实现复杂权限，而是让其他四个人能在统一工程中快速添加页面和接口。

负责：

- 后端 Spring Boot 工程初始化。
- 前端 Vue 3 工程初始化。
- 登录、JWT、基础用户角色。
- 统一异常、返回体、分页、跨域。
- 前端后台布局、菜单、路由守卫和 Axios 封装。
- 合并公共代码并解决公共文件冲突。

不负责：

- 商品、采购、销售、库存业务代码。
- 复杂按钮级权限和多门店数据权限。
- 通用工作流和复杂审计系统。

### 4.3 后端任务清单

- 创建 Maven 工程和基础依赖。
- 配置 MySQL、MyBatis-Plus、Jackson、Validation。
- 创建 `Result<T>` 统一返回对象。
- 创建 `PageResult<T>` 分页对象。
- 创建 `BusinessException` 和全局异常处理器。
- 配置参数校验错误返回。
- 创建 JWT 工具类、认证过滤器和 Spring Security 配置。
- 密码使用 BCrypt 保存，不保存明文密码。
- 创建登录、退出、获取当前用户接口。
- 创建用户 CRUD、启用/禁用、重置密码接口。
- 创建角色 CRUD 和用户分配角色接口。
- 创建简单日志注解或拦截器；来不及时只记录登录及新增、修改、删除操作。
- 配置 OpenAPI/Knife4j 页面。
- 配置开发环境跨域。
- 给其他成员提供 Entity、Controller、Service、Mapper 的示例模块。

## 6. 成员 3：销售、收款和销售报表

### 6.1 职责边界

负责销售单从草稿到审核、完成的基础流程，以及客户收款和销售明细报表。销售出库和实际库存扣减由成员 5 负责。

销售单只需要支持一次性全部出库，不做分批发货、价格策略、优惠券、发票和复杂退货。

### 6.3 后端任务清单

- 创建销售单和明细表、收款单表。
- 后端生成销售单号，例如 `SO202607170001`。
- 新增销售单时后端重新计算每行金额和总金额，不能完全信任前端总额。
- 草稿销售单支持修改和删除。

  6. 将销售单改为 COMPLETED。

| 第 7 天 | 部署最终版本、备份数据库、走完整演示流程、输出部署说明 |

### 8.8 验收条件

- 已审核采购单能生成入库单。
- 确认入库后库存增加并产生一条流水。
- 已审核销售单能生成出库单。
- 库存不足禁止出库。
- 确认出库后库存减少并产生一条流水。
- 重复点击确认不会重复改变库存。
- 首页可以看到主要统计数据。
- 系统能在服务器上通过浏览器访问。

### 8.9 可直接给 AI 的提示词骨架
