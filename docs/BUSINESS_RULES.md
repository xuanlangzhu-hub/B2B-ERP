# 核心业务状态与记账规则

## 单据状态

| 单据 | 状态 | 允许操作 |
| --- | --- | --- |
| 销售单、采购单 | `DRAFT` | 修改、删除、审核、取消 |
| 销售单、采购单 | `APPROVED` | 生成出入库单、取消 |
| 销售单、采购单 | `PARTIALLY_OUTBOUND` / `PARTIALLY_INBOUND` | 继续出入库 |
| 销售单、采购单 | `COMPLETED` | 生成应收或应付，不允许再次出入库 |
| 入库单、出库单 | `DRAFT` | 确认、取消 |
| 入库单、出库单 | `CONFIRMED` | 已记库存，不允许重复确认 |
| 销售退货单、采购退货单 | `DRAFT` | 审核、取消 |
| 销售退货单、采购退货单 | `APPROVED` | 生成退货入库单或退货出库单 |
| 销售退货单、采购退货单 | `COMPLETED` | 已记库存并冲减往来，终态 |
| 销售退货单、采购退货单 | `CANCELLED` | 终态，不允许再次操作 |
| 收款单、付款单 | `DRAFT` | 确认、取消 |
| 收款单、付款单 | `CONFIRMED` | 已记资金和往来，可执行冲销取消 |
| 收款单、付款单 | `CANCELLED` | 终态，不允许再次操作 |
| 库存盘点单 | `DRAFT` | 已生成账面库存快照，可以开始或取消 |
| 库存盘点单 | `COUNTING` | 可以录入实盘数量并提交审核 |
| 库存盘点单 | `APPROVED` | 待审核入账，不允许再修改实盘数量 |
| 库存盘点单 | `COMPLETED` | 已更新库存并生成盘盈盘亏流水，终态 |
| 库存盘点单 | `CANCELLED` | 已取消，不影响库存 |
| 库存调拨单 | `DRAFT` | 可以审核、取消或删除，不影响库存 |
| 库存调拨单 | `APPROVED` | 已校验调出仓库存，可以执行调拨或取消 |
| 库存调拨单 | `COMPLETED` | 已同时更新调出仓和调入仓并生成成对流水，终态 |
| 库存调拨单 | `CANCELLED` | 已取消，不影响库存 |
| 库存调整单 | `DRAFT` | 可以审核、取消或删除，不影响库存 |
| 库存调整单 | `COMPLETED` | 已完成报损或库存增加并生成流水，终态 |
| 库存调整单 | `CANCELLED` | 已取消，不影响库存 |
| 借入借出单 | `DRAFT` | 可以审核、取消或删除，不影响库存 |
| 借入借出单 | `APPROVED` | 已完成首次借入或借出记账，等待归还 |
| 借入借出单 | `PARTIALLY_RETURNED` | 已归还部分商品，可以继续归还剩余数量 |
| 借入借出单 | `COMPLETED` | 所有商品均已归还，终态 |
| 借入借出单 | `CANCELLED` | 已取消，不影响库存 |

## 核心规则

1. 采购入库全部完成时，采购单变为 `COMPLETED` 并生成一笔应付。
2. 销售出库全部完成时，销售单变为 `COMPLETED` 并生成一笔应收。
3. 同一业务来源只能生成一笔应收或应付，重复调用必须保持幂等。
4. 收付款单创建后为草稿，不立即影响账户余额。
5. 确认收款时，按业务日期从早到晚自动核销该客户未结应收，剩余金额记为未分配金额。
6. 确认付款时，按业务日期从早到晚自动核销该供应商未结应付，剩余金额记为未分配金额。
7. 确认收付款必须同时更新账户余额、往来余额、订单结算状态和资金流水，并处于同一事务。
8. 已确认收付款取消时不删除原流水，而是生成反向流水并恢复账户及往来余额。
9. 所有财务数据读取和写入必须校验当前 `enterpriseId`。
10. 金额统一使用 `BigDecimal`，库存数量统一使用四位小数。
11. 只有已完成的销售单或采购单可以发起退货，同一原订单同时只允许存在一张未取消退货单。
12. 销售可退数量为“已出库数量 - 已退数量”，采购可退数量为“已入库数量 - 已退数量”，后端必须再次校验。
13. 销售退货审核后生成退货入库单；确认入库后增加库存、写入 `SALES_RETURN_IN` 流水、回写退货数量并冲减应收。
14. 采购退货审核后生成退货出库单；确认出库后减少库存、写入 `PURCHASE_RETURN_OUT` 流水、回写退货数量并冲减应付。
15. 销售退货按原销售出库成本回库，不使用销售价格计算库存成本；采购退货出库按当前移动平均成本扣减库存。
16. 当前一周版本只自动处理不超过未收/未付余额的退货。涉及已收款退款或已付款退款时拒绝自动完成，留待退款流程实现。
17. 同一仓库同一时间只允许存在一张草稿、盘点中或待审核的盘点单。
18. 创建盘点单时保存仓库当前库存作为账面快照；审核前若账面库存已经变化，必须取消并重新创建盘点单，不能直接覆盖库存。
19. 盘点审核时按实盘数量更新库存余额，盘盈生成 `COUNT/IN` 流水，盘亏生成 `COUNT/OUT` 流水；无差异商品不生成流水。
20. 实盘数量不得小于库存锁定数量，盘点审核必须使用数据库行锁并在同一事务内完成。
21. 调拨单的调出仓和调入仓不得相同；审核时校验调出仓可用库存，执行时再次加锁校验，防止并发导致负库存。
22. 调拨完成必须在同一事务内扣减调出仓、增加调入仓，并生成数量相同的 `TRANSFER_OUT/OUT` 与 `TRANSFER_IN/IN` 成对流水。
23. 调拨进入目标仓时沿用调出仓成本，并按目标仓原库存金额和本次调入金额重新计算移动平均成本。
24. 库存调整分为 `INCREASE` 和 `DECREASE`；报损属于减少调整，后端必须校验调整后现存量和可用量均不小于零。
25. 库存增加调整按照原库存金额与本次增加金额计算移动平均成本；减少调整按照当前移动平均成本扣减库存金额。
26. 调拨执行和库存调整审核都必须锁定单据及相关库存行，并通过终态校验保证重复请求不会重复记账。
27. 借出单审核后减少库存并生成 `BORROW_OUT/OUT` 流水，借出归还增加库存并生成 `BORROW_OUT_RETURN/IN` 流水。
28. 借入单审核后增加库存并生成 `BORROW_IN/IN` 流水，借入归还减少库存并生成 `BORROW_IN_RETURN/OUT` 流水。
29. 每条借用明细单独累计已归还数量，单次归还不得超过未归还数量；全部归还后单据才进入 `COMPLETED`。
30. 借出及借入归还的扣库环节必须再次锁定并校验可用库存，整个单据和库存流水更新处于同一事务。
31. 预计归还日期不得早于借用日期；超过预计日期且仍处于借用中或部分归还的单据标记为逾期。
32. 商品属性、商品标签和客户标签均按企业隔离；保存商品或客户时，后端必须验证所选配置属于当前企业。
33. 选择型商品属性只能引用该属性下仍处于启用状态的属性值；文本型和数值型属性保存自定义值，数值型必须能够转换为有效数值。
34. 商品基础信息与商品标签、属性关系必须在同一事务中保存；客户基础信息与客户标签关系同样必须在同一事务中保存。
35. 已被商品或客户使用的属性、属性值和标签不能直接删除，必须先解除业务关系。
36. 销售报表只统计状态为 `COMPLETED` 的销售单；草稿、审核中、部分出库和已取消单据不计入销售业绩。
37. 销售净额等于销售明细金额减去已完成销售退货金额；销售成本取真实销售出库流水成本并扣除已完成退货回库成本，毛利润等于销售净额减销售成本。
38. 报表必须按当前企业隔离，汇总卡片基于全部筛选结果计算，不能只累加当前分页；基础资料后续停用或软删除不能导致历史完成单据从报表消失。
39. 采购报表只统计状态为 `COMPLETED` 的采购单；草稿、审核中、部分入库和已取消单据不计入采购统计。
40. 采购净数量等于完成入库数量减已完成采购退货数量，采购净额等于采购明细金额减已完成采购退货金额。
41. 采购明细、商品汇总、供应商汇总和分类汇总必须共用相同的日期及筛选口径，汇总卡片不得仅统计当前分页。

## 订单结算状态

| 状态 | 含义 |
| --- | --- |
| `UNPAID` | 尚未发生有效收付款 |
| `PARTIALLY_PAID` | 已收或已付金额大于 0，但仍有余额 |
| `PAID` | 应收或应付余额为 0 |
