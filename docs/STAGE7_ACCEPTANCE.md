# 第七阶段完成与验收记录

## 已完成范围

- 库存报表：库存余额、库存预警、库存流水、分类/仓库/商品/方向/来源/日期筛选及汇总。
- 财务报表：客户对账单、供应商对账单、期初/本期发生/退货/收付款/期末余额及往来明细。
- 资金管理：其他收入、其他支出，确认后更新账户并生成资金流水，取消后自动回冲。
- 系统设置：企业信息、员工启停和默认门店、角色菜单树授权、企业隔离操作日志。
- 个人中心：个人资料、修改密码、可访问门店和顶栏门店切换。
- 消息通知：草稿、发布、列表、未读数、阅读记录。
- 页面统一：固定全屏主布局、深色侧栏、面包屑、统一卡片/表格/工具栏/弹窗和常见分辨率适配。
- 登录注册：原型式城市背景登录页、两步注册页，以及企业/门店/仓库/管理员的注册初始化后端。
- 独立单据页：销售单、采购单、销售退货、采购退货、盘点、调拨、报损/调整、借出和借入均已拆分列表、新增与详情路由。
- 演示数据：新增系统通知、新菜单权限和管理员门店访问关系。

## 数据库脚本

在 V1 到 V8 后继续执行：

```text
sql/V9__system_permissions_and_demo.sql
```

脚本可重复执行，会创建通知阅读表，并补齐其他收支、企业信息、消息通知菜单与角色权限。

## 自动检查结果

- `mvn test`：通过。
- `npm run build`（包含 Vue TypeScript 检查）：通过。
- 本地 MySQL 真实接口：登录、菜单、员工、角色、企业信息、通知、库存余额/流水报表、客户/供应商对账单均通过。
- 其他收支闭环：新增 → 确认 → 取消 → 删除通过；确认与回冲资金流水净额为 0，测试数据已清理。

## 演示建议

1. 使用 `admin / Admin@123456` 登录。
2. 在顶栏切换门店并查看未读通知。
3. 进入库存报表切换“库存余额 / 库存预警 / 库存流水”。
4. 进入财务报表查看客户或供应商对账单，再打开往来明细。
5. 新建一笔其他收入，依次演示确认、资金流水和取消回冲。
6. 在角色管理中演示菜单树授权，在员工管理中演示角色和门店配置。

## 暂不作为本轮验收内容

- 短信、打印模板、组合商品、复杂绩效。
- 像素级复刻原型；本轮先保证页面结构、交互和风格统一，后续按实际截图返工差异。
